# web — Web 管理端（Vue + ECharts）

> 本目录是给「后台管理人员」用的网页系统。顾客和骑手用手机 App，而商家、数据分析师、监管人员则坐在电脑前用浏览器打开这个网页来工作。

---

## 一、这个文件夹是干什么的？

`web` 存放**网页版管理后台**的全部代码。它运行后是一个网站，在浏览器里打开（通常是 `http://localhost:5173`），不同身份的人登录后看到不同的功能页面。

为什么管理端要做成网页、而不是 App？因为后台工作（看报表、处理大量订单、维护数据）在大屏幕电脑上操作更方便，而网页天然适合电脑，打开即用、无需安装。

打个比方：
- 移动端 App = 面向**普通用户**的随身工具（点餐、送餐）
- web 管理端 = 面向**内部工作人员**的办公系统（管理、分析、监督）

---

## 二、它在整个项目架构里的位置

```
┌─────────────┐     ┌─────────────────────┐
│ 移动端 App   │     │ web/  ← 你在这里       │
│ (mobile/)   │     │ Vue 网页管理后台        │
│ 顾客 / 骑手  │     │ 商家 / 分析师 / 监管    │
└──────┬──────┘     └──────────┬──────────┘
       │                       │
       │   都通过 HTTP 请求 (JSON) │
       └───────────┬───────────┘
                   ▼
         ┌───────────────────┐
         │   backend/ (后端)   │
         └─────────┬─────────┘
                   ▼
         ┌───────────────────┐
         │   MySQL 数据库      │
         └───────────────────┘
```

**关键点：web 端自己不存数据，也不直接连数据库**。它做的事是：把界面画得漂亮 → 用户点按钮 → 向 backend 发请求 → 拿到数据 → 用图表/表格展示出来。所有数据都来自 backend 提供的 API。

## 三、这里面应该放什么（推荐目录结构）

用 Vue 3 创建项目后，大致结构如下（脚手架会自动生成一部分，你主要在 `src/` 里干活）：

```
web/
├── README.md            # 本文件
├── package.json         # 项目依赖清单（相当于后端的 requirements.txt）
├── vite.config.js       # 构建工具配置
├── index.html           # 网页入口
└── src/                 # 你写的代码几乎都在这里
    ├── main.js          #   程序入口
    ├── App.vue          #   根组件
    ├── api/             #   封装对 backend 的所有请求（在一处统一管理）
    │   └── request.js
    ├── router/          #   页面路由：哪个网址对应哪个页面
    ├── views/           #   一个个完整页面
    │   ├── Login.vue          # 登录页
    │   ├── MerchantBoard.vue  # 商家：订单处理看板
    │   ├── Analytics.vue      # 分析师：多维数据分析 + ECharts 图表
    │   └── Regulator.vue      # 监管：只读监管视图 + 危险路段维护
    └── components/      #   可复用的小部件（按钮、图表卡片等）
```

> `node_modules/`（安装的第三方库，体积巨大）已在根目录 `.gitignore` 排除，不上传 GitHub。队友拉取代码后，自己运行 `npm install` 会根据 `package.json` 重新下载。

## 四、这里负责实现哪些评分要求

| 评分要求 | 在 web 端如何实现 |
| --- | --- |
| Web 客户端 | 整个 web 目录就是 |
| 多角色不同界面 | 商家 / 分析师 / 监管 三套不同页面 |
| 界面美观、便捷、易用 | Vue 组件 + 统一样式设计 |
| 数据分析与多维可视化 | `Analytics.vue` 用 ECharts 画图：按时间/地区/品类的销量分析 |
| 伦理（交通安全） | `Regulator.vue` 维护 risk_zones 危险路段表 |
| AI 推荐/预测展示 | 把后端 AI 接口的结果用图表呈现 |

**三类用户的功能区分：**
- **商家**：订单处理看板——接单、出餐、查看订单状态流转。
- **分析师**：多维数据分析——销量按不同维度（时间/地区/品类）切片，用 ECharts 折线图、柱状图、饼图展示。这是「数据可视化」得分的核心。
- **监管**：只读监管视图（不能改业务数据）+ 维护危险路段表，体现「伦理-交通」要求。

## 五、新手怎么开始（Phase 4 才正式动手）

1. **确保电脑装了 Node.js**（去 nodejs.org 下载 LTS 版）。检查：
   ```bash
   node -v
   ```
2. **创建 Vue 项目**：
   ```bash
   npm create vue@latest
   ```
   按提示选项目名等，然后：
   ```bash
   cd 项目名
   npm install
   npm run dev
   ```
3. 浏览器打开终端显示的地址（通常 `http://localhost:5173`），看到欢迎页就成功了。
4. **安装图表库 ECharts**（做可视化要用）：
   ```bash
   npm install echarts
   ```

> 现在是 Phase 0，本目录暂时只放这份 README。Web 端是 Phase 4 的任务，需要先等 backend 把 API 做出来（或用 Mock 接口）才能联调。
