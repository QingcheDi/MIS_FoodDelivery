# db — 数据库（MySQL 主库 + SQLite 原型）

> 本目录是整个系统的「仓库」相关文件存放处。它不存放运行中的数据本身，而是存放**如何建立和初始化数据库的脚本、设计图和文档**。真正的数据躺在 MySQL 服务里。

---

## 一、这个文件夹是干什么的？

`db` 存放和**数据库设计**有关的一切：建表脚本（SQL 文件）、ER 关系图、数据字典、初始测试数据等。

要分清两个概念：
- **数据库服务（MySQL）**：在你电脑上运行的程序，真正存着用户、订单等数据。它不在这个文件夹里。
- **db/ 目录**：存放「怎么把那个数据库建起来」的图纸和脚本。新队友拿到这些脚本，在自己电脑上一跑，就能复刻出一模一样的数据库结构。

打个比方：db/ 目录 = **建筑图纸**，MySQL 服务 = **照图纸盖好的真实仓库**。

---

## 二、它在整个项目架构里的位置

```
   backend/ (后端)
        │
        │ 后端用 SQL 读写数据
        ▼
┌───────────────────────┐
│  MySQL 数据库服务（运行中） │  ← 真实数据在这里
└───────────────────────┘
        ▲
        │ 用 db/ 里的脚本建立 & 初始化
        │
┌───────────────────────┐
│  db/  ← 你在这里         │
│  - schema.sql 建表脚本   │
│  - ER 图 / 数据字典      │
│  - 初始测试数据          │
└───────────────────────┘
```

**两种数据库工具的分工**（项目要求）：
- **DB Browser for SQLite**（老师指定）：用来**快速画原型**、验证表结构想法，也用作移动端的本地缓存。轻量、零配置，适合新手先练手。
- **MySQL**：**正式主库**，最终系统真正使用的数据库。要求表设计规范化、并实现数据库层面的权限控制。

流程：先在 DB Browser 里快速试 → 想清楚后，把结构正式落到 MySQL。


## 四、核心表有哪些（方案规划）

这套系统至少需要以下表，覆盖所有评分要求：

| 表名 | 作用 |
| --- | --- |
| `users` | 用户（密码用 bcrypt 加密存储，绝不明文） |
| `roles` | 角色：顾客 / 骑手 / 商家 / 分析师 / 监管 |
| `permissions` | 权限项 |
| `menu_items` | 菜品 |
| `orders` | 订单（含状态、金额、关联顾客与骑手） |
| `order_status_log` | 订单状态变更流水（可追溯每次状态变化） |
| `delivery_photos` | 配送照片/语音的元数据（存路径，文件本身在磁盘 = 非结构化存储） |
| `risk_zones` | 危险路段（伦理-交通功能用） |
| `carbon_log` | 碳排放记录（伦理-环保功能用） |

## 五、这里负责实现哪些评分要求

| 评分要求 | 在 db 如何实现 |
| --- | --- |
| MySQL 数据库服务器 | `schema.sql` 建出规范化的库 |
| 数据库表合理设计 | 规范化设计、合理的主外键与索引 |
| 数据库权限分级（RBAC） | `rbac.sql`：给分析师建只读账号、给监管建只读视图账号 |
| 密码加密 | `users` 表的密码字段存 bcrypt 哈希值 |
| 非结构化存储 | `delivery_photos` 存路径，文件本体在磁盘 |
| 伦理（环保/交通） | `carbon_log` 与 `risk_zones` 两张表支撑 |

## 六、新手怎么开始（Phase 1 才正式动手）

1. **装 MySQL** + 一个客户端（MySQL Workbench 或 DBeaver）。
2. **装 DB Browser for SQLite**（老师指定，先用它练手画原型）。
3. Phase 1 时，先在 DB Browser 里把表结构试出来，确认字段无误后，写成 `schema.sql`，在 MySQL 里执行建出正式库。
4. **数据库层 RBAC** 的核心命令长这样（Phase 1 实现）：
   ```sql
   -- 建一个只能读数据的分析师账号
   CREATE USER 'analyst'@'localhost' IDENTIFIED BY '密码';
   GRANT SELECT ON fooddelivery.* TO 'analyst'@'localhost';
   ```
   这样即使分析师账号泄露，别人也只能看、不能改，体现「分级权限控制」。

> 现在是 Phase 0，本目录暂时只放这份 README。Phase 0 最该先产出的就是 **数据字典**（`data_dictionary.md`）——它是后面所有建表和写代码的蓝图。
