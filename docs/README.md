# docs — 项目文档

> 本目录存放整个项目的「说明书」和「设计图纸」。它不含任何运行代码，但却是 Phase 0 最重要的产出地——这里的文档一旦定好，前端、后端、移动端就能各自照着图纸并行开工，互不阻塞。

---

## 一、这个文件夹是干什么的？

`docs` 存放**所有设计文档与团队约定**：系统架构图、数据字典、API 契约、错误码规范、成员分工表等。

为什么文档这么重要？因为我们是多人协作的团队项目。如果不先把「接口长什么样、数据怎么存、谁负责哪块」写清楚，就会出现：后端返回的数据格式前端看不懂、两个人改同一处代码起冲突、答辩时说不清自己做了什么……

打个比方：docs/ = **施工前的设计蓝图和施工合同**。先把图纸画好、责任划清，盖楼时各工种才能同时开工不打架。

---

## 二、它在整个项目架构里的位置

docs 不属于「运行系统」的任何一层，它**横跨并指导所有层**：

```
        ┌─────────────────────────────┐
        │   docs/  ← 你在这里            │
        │   架构图 · 数据字典 · API 契约   │
        │   错误码规范 · 成员分工         │
        └──────────────┬──────────────┘
                       │ 指导 & 约束
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   ┌─────────┐   ┌─────────┐   ┌─────────┐
   │ mobile/ │   │  web/   │   │backend/ │
   └─────────┘   └─────────┘   └────┬────┘
                                    ▼
                              ┌─────────┐
                              │   db/   │
                              └─────────┘
```

- **API 契约**告诉 mobile、web、backend 三方：接口的网址、要传什么、会返回什么。三方照着它各写各的。
- **数据字典**告诉 backend 和 db：每张表有哪些字段、什么类型。
- **成员分工**告诉所有人：谁负责哪个模块（直接关系答辩得分）。

## 四、几份核心文档分别是什么

**1. 系统架构图（architecture.png）**
一张图说明：移动端、Web 端、后端、数据库之间怎么连、数据怎么流。用 draw.io 或 Excalidraw 画完导出图片即可，不求美观，求清晰。

**2. 数据字典（data_dictionary.md）⭐ 最该先做**
把每张表的每个字段列清楚：字段名、类型、是否可空、默认值、外键、说明。这是 db 建表和 backend 写模型的共同蓝图。

**3. API 契约（api_contract/）⭐ 最该先做**
把所有接口定死，例如：
```
POST /api/auth/login        登录
  请求: { "username": "string", "password": "string" }
  成功 200: { "token": "jwt...", "role": "customer", "user_id": 1 }
  失败 401: { "code": "AUTH_FAILED", "msg": "用户名或密码错误" }
```
推荐用 **Apifox**（中文友好）把接口建好，并开启 **Mock**——这样前端不用等后端，就能拿到假数据先开发。

**4. 错误码规范（error_codes.md）**
统一约定所有接口返回的格式（例如都包成 `{ code, msg, data }`）和错误码含义，避免各写各的。

**5. 成员分工表（team_roles.md）⭐ 关系答辩得分**
老师答辩时会逐个问「你负责哪块、代码在哪、怎么实现的」。现在就把模块责任人定死写进来，例如：

| 成员 | 负责模块 |
| --- | --- |
| A | 后端认证 + 安全 |
| B | 后端订单 + 上传 |
| C | 移动端（顾客 + 骑手） |
| D | Web 管理端 + 可视化 |
| E | AI + 伦理模块 |

## 五、Phase 0 的验收标准

当 docs 里的文档齐全到「**任意一名成员能照着 API 契约，不问别人就独立开始写自己那块，且 Mock 接口能返回假数据**」时，Phase 0 就算完成，可以进入 Phase 1（建数据库）。

> 现在正处于 Phase 0。下一步最高优先级：先把 **数据字典** 和 **API 契约** 这两份做出来——它们是动工前收益最高的产出。
